Purpose

Why this tool exists — to provide a trusted, auditable path for users to download approved applications without confusing or unsafe prompts. The project focuses on verification, clear guidance, and graceful failure states for novice and power users.

Intended Audience

Event organizers, enterprise deployers, and technical communities that need a branded, secure download surface with easy verification and reproducible releases.

How To Use

Download and Verify

  1. Choose the correct platform build from the download button.
  2. Download the installer file and the matching .sha256 checksum file or copy the shown SHA256 string.
  3. Verify the checksum locally before running the installer using your platform tools or our built-in verify utility.

Run Installer Safely

  1. Close unrelated apps that may interfere with installation.
  2. Run the installer with administrator privileges only if prompted and only when the signature and checksum match.
  3. If using Windows, prefer the signed MSI/EXE distribution over unsigned ZIP or script packages.

About the Don't Install Window

You may see a "Don't Install Window" or a similar warning during or after download. This is usually a protective operating system or browser prompt, not an indication that the tool is malicious. Common causes include mis-signed installers, mismatch with system policies, or the browser's smart screening heuristics.

Why it appears

  • Installer signature not recognized by the OS because a timestamp or certificate chain is missing.
  • Checksum mismatch from incomplete or corrupted downloads.
  • Browser or OS heuristic flagged the download as uncommon or non-popular.
  • Enterprise or local security policies block new installers by default.

How to prevent false Dont Install prompts

  • Always verify SHA256 checksum before opening any installer file.
  • Use the signed installer packages we provide and validate the certificate chain in your OS.
  • Download from official domains and avoid mirror sources unless authorized.
  • If an enterprise policy blocks installers, consult your IT admin to whitelist the signed artifact.
  • When a prompt appears, read it fully: if the checksum and signature match our published values, proceed; otherwise abort and contact support.

If the prompt persists

Capture the exact prompt text and installer checksum, then open a support ticket. Do not override security dialogs unless you have validated the file and trust the source.
